Discussion

Interpretation of Results

The gradient descent algorithm successfully minimized the MSE loss function, finding parameters $\theta_0 \approx 7$ and $\theta_1 \approx 3.5$ — very close to the true data-generating parameters ($\theta_0^* = 7$, $\theta_1^* = 3.5$). The small residual error arises from the Gaussian noise added to the dataset, which is irreducible.

The convergence plot shows the classic behavior of gradient descent on a strongly convex objective: fast initial descent followed by progressively smaller updates as the algorithm approaches the optimum. The majority of loss reduction occurs within the first 100 iterations.

Effect of Hyperparameters

Learning Rate α

A learning rate of $\alpha = 0.01$ was chosen after testing. Values that are too large ($\alpha > 0.1$) cause divergence; values too small ($\alpha < 0.001$) converge slowly. $\alpha = 0.01$ provides a good balance.

Number of Iterations

500 iterations is more than sufficient for this problem — the loss plateaus around iteration 150. For larger datasets or higher-dimensional problems, more iterations or early stopping would be needed.

Initialization

We initialized $\theta = \mathbf{0}$. Because MSE is convex, the choice of initialization only affects the trajectory, not the final solution.

Dataset Size

With 80 samples and a signal-to-noise ratio typical for regression problems, the estimator is statistically consistent and the gradient estimates are stable.

Limitations

The current implementation uses a fixed learning rate. More advanced variants — such as Adam, RMSProp, or learning-rate scheduling — can converge faster or more robustly on ill-conditioned problems.

Additionally, batch gradient descent (used here) computes the gradient over the entire dataset at every step, which is expensive for large datasets. Stochastic Gradient Descent (SGD) or mini-batch GD offer better scalability at the cost of noisier updates.

Conclusions

This project demonstrated the full pipeline of mathematical optimization applied to a regression problem:

  1. We formulated the problem as minimizing MSE — a smooth, convex quadratic loss.
  2. We derived the gradient analytically and implemented it in pure NumPy.
  3. Gradient descent converged to parameters very close to the ground truth within 500 steps.
  4. Three visualizations confirmed convergence, fit quality, and the trajectory on the cost surface.

The experiment confirms that gradient descent is a reliable, interpretable algorithm for MSE minimization, and that understanding the mathematical structure of the loss function is key to choosing appropriate hyperparameters.
